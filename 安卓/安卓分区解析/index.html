安卓分区

Boot 分区

        Boot 分区存放启动和引导文件，类似于 windows 系统中的 esp 分区。它包含操作系统的 kernel（内核）和 ramdisk（虚拟内存盘）。magisk 对手机进行 root 就是对 Boot 分区进行修改。Boot 分区损坏，开机就会卡在 logo 界面。

        | 虚拟内存盘：将通过软件将一部分内存（RAM）模拟为硬盘来使用的一种技术，可以极大的提高在其上进行的文件访问的速度。

System 分区

        System 分区包含操作系统和系统预装软件。它是系统分区和刷机的主要对象。擦除该分区不会导致无法启动，但是如果该分区损坏，就会开在开机动画界面。

Vendor 分区

        Vendor 分区包含开发厂商定制的应用和库文件，属于厂商夹带的私货，比如小米手机的小米音乐，小米视频等。现在厂商大多把它放到 system 分区中。

Data 分区

        又叫 Userdata 分区，挂在在/data 目录下。此分区包含了用户的数据信息，如：联系人、短信、设置、用户安装的程序的，擦除此分区，相当于手机恢复出厂设置。不会影响手机开机。Andorid11 后，对 data 分区进行了加密。

Cache 分区

        Cache 分区是安卓系统的缓存区，保存系统最常访问的数据和应用程序。擦除这个分区，不会影响个人数据，只是删除了这个分区中已经保存的缓存内容，缓存内容会在后续手机使用过程中重新自动生成。

Recovery 分区

        恢复分区，相当于 Windows 的 pe 系统。刷机必备。一般手机自带的 recovery 分区功能较少，所以我们会刷入第三方 rec。现在的一些手机将 rec 分区合并到了 boot 分区中，那么相应的，刷机也自然有不用 rec 的办法。

        我们将用 rec 的刷机方式叫做卡刷。

Modem 分区

        Modem 分区，又或者叫做 Radio 分区。保存手机的基带和芯片等。通俗的来说，就是手机的联网，打电话，发短信等与电信网络交互的行为都是靠这个分区。

Misc 分区

        Miscellaneous，杂项。这个分区存放了一些乱七八糟的设置，也是非常重要的一个分区。Misc 分区损坏，会导致手机一些特定功能无法正常工作。

Hboot 分区（bootloader）

        Hboot 分区用来保存 bootloader。这个 bootloader 相当于电脑的 bios/uefi，是根植于手机硬件的。也是刷机党们最常用的模式。bootloader 一般来说不会损坏，这也是手机变砖前的倒数第二道防线。如果手机连 bootloader 都进不去，那只能考虑 9008 了。如果 9008 都不行，那就用来砸人吧。

Bootloader 和 Fastboot

        一开始刷机的时候，总是搞不清楚这两个东西。有说进入 bootloader 的，有说进入 fastboot 模式的。那么在长时间的刷机过程中呢，我对这两个东西有了一些自己的理解，希望可以帮到大家区分这两个模式。

        首先，刷机教程中，无论是 bootloader 还是 fastboot 模式，其实都是一个东西，比如下图中小米的兔子。

        我个人认为呢，这个应该叫做 bootloader，使用的是 fastboot 命令，所以可以叫做 fastboot 模式。他们的区别在于，bootloader 是一段引导程序，而 fastboot 是在这个程序中可以使用的命令。在这个模式下刷机，我们称之为线刷。线刷也是目前没有 rec 分区的手机最常用的刷机方法。

安卓系统启动分析

        对于安卓启动的过程做一个大致的分析，不涉及开发层面。

普通手机

  1. 按下电源，bootloader 开始引导启动。

  2. bootloader 初始化设备，引导系统内核。可进入 bootloader，开始线刷。

  3. 在 bootloader 准备内存（ram）空间。可进入 rec 分区，开始卡刷

  4. 进入 boot 分区，启动系统。

        从上面的启动过程可以看出，fastboot 比 rec 要更加的底层。所以当 rec 不好用时，我们依然可以尝试进入 fastboot。

A/B 分区手机

        A/B 分区与普通的情况不同之处就在于有两个分区。假设，正常情况下我们使用 A 分区的 systemA，那么，在升级系统或者安装刷机包时，就会安装到 B 分区的 systemB。

        这样的设计，在 ota 升级给予了极大的方便。我们可以在 systemA 正常使用手机，后台把系统刷入 systemB。等到升级完成，就从 Slot B 进入 B 分区。同系统的升级，对于用户个人数据，即 data 分区是没有影响的。

        但是 A/B 分区的缺点也是同样明显的，占用双倍的 system 空间，非常的浪费。因此出现了 VA/B 分区（Virtual A/B System Updates）。结合普通手机和 A/B 分区的特点，将 boot 分区和 system 分区分出虚拟的 A/B 分区。相同文件公用，不同文件存放到虚拟区。

总结

        A/B 分区手机相对来说不是那么好刷，没有了 rec 分区，第三方的 rec 必须刷两遍进行固定。建议在刷机之前，要先了解如何备份，救砖。刷机出现变砖是常有的事情，遇到不要慌。还有就是不要用主力机，除非你已经非常非常的熟练了。
